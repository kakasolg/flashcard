# 💡 지능형 플래시카드 앱 - 개발 계획

## 🎯 프로젝트 목표

사용자에게 최적화된 학습 경험을 제공하는 지능형 플래시카드 애플리케이션 개발. LLM, LangGraph, MongoDB, 벡터 저장소 등의 기술을 활용하여, 대화형 카드 생성, 간격 반복 학습, 의미론적 검색, 개인별 학습 분석 및 피드백 기능을 구현한다.

## 🗺️ 단계별 개발 계획

### Phase 1: 코어 플래시카드 시스템 및 데이터 기반 구축 🧱

> 목표: 기본적인 플래시카드 생성, 저장, 복습 기능 구현 및 데이터 모델 정의

* **Task 1.1**: 프로젝트 환경 설정
    * Git 저장소 초기화 및 관리 전략 수립.
    * `uv`를 사용한 Python 가상 환경 설정 (예: Python 3.12).
    * 기본적인 프로젝트 폴더 구조 생성.
* **Task 1.2**: 핵심 데이터 모델 정의
    * Python `dataclasses` 활용: `FlashCard`, `CardDifficulty`, `ChunkingProfile`, `Loci` 등 초기 버전 정의.
    * 데이터 간의 관계 초기 설계.
* **Task 1.3**: MongoDB 설정 및 기본 CRUD 연동
    * MongoDB 인스턴스 준비 (로컬 또는 클라우드).
    * Python에서 MongoDB 연결 및 플래시카드 데이터 생성, 읽기, 수정, 삭제 (CRUD) 기능 구현 (수동 입력 방식).
* **Task 1.4**: 간격 반복 학습(SRS) 엔진 구현
    * SM-2 또는 유사 알고리즘 기반의 SRS 로직 개발 (`SpacedRepetitionEngine` 클래스).
    * 카드 난이도 평가에 따른 다음 복습 일정 계산 로직 구현.
* **Task 1.5**: 기본 CLI (Command Line Interface) 개발
    * 텍스트 기반으로 플래시카드 생성, 목록 보기, 학습 세션 시작.
    * 학습 세션 중 카드 앞면 표시, 사용자 답변 입력, 정답 확인, 난이도 수동 입력 기능.

---

### Phase 2: LLM 연동을 통한 카드 생성 및 기본 상호작용 🤖

> 목표: LLM을 활용하여 사용자와의 대화를 통해 플래시카드 자동 생성 기능 구현.

* **Task 2.1**: LLM API 연동 준비
    * 사용할 LLM 선정 (예: OpenAI GPT 모델) 및 API 키 발급.
    * Python에서 LLM API 호출 및 응답 처리 기본 모듈 개발.
* **Task 2.2**: 플래시카드 생성을 위한 초기 프롬프트 설계
    * LLM이 사용자에게 질문을 던지고, 그 답변을 기반으로 플래시카드의 `front`와 `back` 내용을 생성하도록 하는 프롬프트 개발.
    * 생성된 내용의 형식(예: JSON) 정의.
* **Task 2.3**: LangGraph를 활용한 대화 흐름 관리
    * 플래시카드 생성을 위한 최소한의 다단계 대화 흐름(stateful)을 LangGraph로 구현.
    * 사용자 입력 -> LLM 질문 생성 -> 사용자 답변 -> LLM 카드 생성 -> 확인/저장.
* **Task 2.4**: LLM 생성 카드 MongoDB 저장
    * LLM이 생성한 플래시카드 데이터를 MongoDB에 저장하는 로직 연동.

---

### Phase 3: 벡터 저장소 연동 및 의미론적 검색 기능 추가 🔍

> 목표: 플래시카드 내용의 의미 기반 검색 및 연관 카드 추천 기반 마련.

* **Task 3.1**: 벡터 저장소(Vector Store) 설정
    * ChromaDB, FAISS 등 적합한 벡터 저장소 선택 및 설치/설정.
* **Task 3.2**: 플래시카드 내용 임베딩(Embedding) 기능 구현
    * 선택한 임베딩 모델 (예: OpenAI `text-embedding-ada-002` 또는 오픈소스 Sentence Transformers) 연동.
    * 플래시카드의 `front`, `back` 또는 조합된 텍스트를 임베딩 벡터로 변환하는 로직 개발.
* **Task 3.3**: 벡터 저장소에 플래시카드 임베딩 저장
    * 새로운 플래시카드 생성 시 또는 기존 카드 업데이트 시, 임베딩 벡터를 생성하여 MongoDB ID와 함께 벡터 저장소에 저장/업데이트.
* **Task 3.4**: 기본 의미론적 검색 기능 구현
    * 사용자 검색어나 특정 플래시카드 내용을 기준으로, 벡터 저장소에서 의미적으로 유사한 플래시카드를 검색하여 ID 목록 반환.
    * 반환된 ID를 이용해 MongoDB에서 실제 카드 정보 조회.

---

### Phase 4: 고급 기능 구현 및 사용자 학습 분석 강화 📊

> 목표: 멀티미디어, 기억 용량 테스트, 기억의 궁전, 학습 기록 분석 등 고급 기능 추가.

* **Task 4.1**: Q&A 상호작용 히스토리 로깅
    * 사용자의 모든 학습 세션(질문, 답변, 정답 여부, 선택한 난이도, 응답 시간 등) 상세 정보를 MongoDB의 별도 컬렉션에 기록.
* **Task 4.2**: 학습 히스토리 기반 분석 및 피드백 기능 초안
    * 저장된 Q&A 히스토리를 분석하여 사용자가 자주 틀리거나 어려워하는 카드/개념 식별.
    * (선택적) 간단한 통계 시각화 (`matplotlib` 등 활용).
* **Task 4.3**: 멀티미디어 요소 연동 (`MultimediaProvider` 활용)
    * 플래시카드에 이미지 URL, 오디오 URL, 시각적 단서(텍스트) 필드 추가 및 관리.
    * (초기) 샘플 데이터 또는 URL 직접 입력 방식, (추후) 외부 API 연동 고려.
* **Task 4.4**: 개인별 기억 용량(Chunking) 테스트 및 프로필 관리
    * `ChunkingTester` 로직 구현 및 사용자별 `ChunkingProfile` 업데이트 기능 개발.
    * 테스트 결과를 바탕으로 학습 시 제시되는 정보량 조절 로직 연동 (초기 아이디어).
* **Task 4.5**: 강화된 기억의 궁전 시스템 로직 구체화
    * `EnhancedMemoryPalace` 클래스의 기능들 (로사이 구조 설정, 카드와 로사이 연결, 경로 생성 등) 구현 및 기존 시스템과 통합.

---

### Phase 5: API 개발 및 백엔드 시스템 정제 ⚙️

> 목표: 프론트엔드 연동을 위한 안정적이고 확장 가능한 API 서버 구축.

* **Task 5.1**: RESTful API 설계 및 구현
    * 웹 프레임워크(예: FastAPI, Flask)를 사용하여 주요 기능(카드 CRUD, 학습 세션, SRS 업데이트, 검색, 사용자 정보 등)을 위한 API 엔드포인트 개발.
    * 요청/응답 데이터 형식(Pydantic 모델 등) 정의.
* **Task 5.2**: API 보안 기초 적용
    * (초기) API 키 인증 또는 간단한 토큰 기반 인증 구현.
* **Task 5.3**: 백엔드 코드 리팩토링
    * 모듈화, 가독성, 효율성, 확장성을 고려하여 기존 코드베이스 개선.
    * 오류 처리 및 로깅 강화.

---

### Phase 6: 프론트엔드 UI/UX 개발 💻

> 목표: 사용자가 편리하게 애플리케이션을 사용할 수 있는 웹 인터페이스 개발.

* **Task 6.1**: 사용자 인터페이스(UI) 및 사용자 경험(UX) 설계
    * 주요 화면(카드 목록, 학습 화면, 통계, 설정 등) 와이어프레임 및 목업 디자인.
    * 사용자 흐름 정의.
* **Task 6.2**: 프론트엔드 애플리케이션 개발
    * 웹 프레임워크(예: React, Vue, Svelte) 선택 및 개발 환경 구축.
    * 설계된 UI 컴포넌트 개발.
    * (초기 프로토타입용으로 Streamlit, Gradio 같은 Python 기반 도구도 고려 가능)
* **Task 6.3**: 백엔드 API 연동
    * 프론트엔드에서 백엔드 API를 호출하여 데이터 송수신 및 기능 연동.
* **Task 6.4**: 사용자 인증 및 세션 관리
    * 회원가입, 로그인, 로그아웃 기능 구현.
    * 안전한 세션 관리.

---

### Phase 7: 통합 테스트, 배포 및 지속적 개선 🚀

> 목표: 안정적인 서비스 제공 및 사용자 피드백 기반의 지속적인 업데이트.

* **Task 7.1**: 종합 테스트
    * 단위 테스트, 통합 테스트, E2E(End-to-End) 테스트 케이스 작성 및 수행.
    * 주요 기능 및 예외 상황에 대한 테스트 커버리지 확보.
* **Task 7.2**: 배포 환경 구축 및 CI/CD 파이프라인 설정
    * Docker를 활용한 컨테이너화 (선택적).
    * 클라우드 플랫폼(AWS, GCP, Azure 등) 또는 기타 호스팅 서비스 선택 및 배포.
    * CI/CD (Continuous Integration/Continuous Deployment) 파이프라인 구축 (예: GitHub Actions).
* **Task 7.3**: 초기 버전 배포 (알파/베타)
    * 제한된 사용자 그룹 대상 또는 공개 베타 버전 릴리스.
* **Task 7.4**: 사용자 피드백 수집 및 반복 개선
    * 피드백 채널 마련 및 주기적인 기능 개선, 버그 수정.
    * 성능 모니터링 및 최적화.

## 🔮 향후 고려 사항 (선택적)

* 모바일 애플리케이션 개발 (iOS, Android).
* LLM 기반 고급 튜터링 기능 (심층적인 오답 분석, 맞춤형 학습 경로 제안).
* 사용자 간 플래시카드 공유 및 커뮤니티 기능.
* 게임화(Gamification) 요소 도입.
